# banking-demo Ansible

Provisions and deploys the banking-demo stack on an EC2 instance (Ubuntu 26.04)
created by the Terraform configuration in `../terraform/`.

## Prerequisites

> **Windows users:** Ansible does not run natively on Windows. Use
> [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) and run all
> Ansible commands from inside a WSL Ubuntu shell.

```bash
# inside WSL / Linux / macOS
pip install ansible-core
ansible-galaxy collection install community.general ansible.posix
```

## Workflow

```bash
# 1. Provision EC2 and generate inventory (run from Windows or WSL)
cd infra/terraform
terraform apply

# 2. Deploy — one command, no manual setup needed
cd infra/ansible
chmod +x deploy.sh
./deploy.sh
```

`deploy.sh` handles all WSL friction automatically:
- Copies your `.pem` key from the Windows filesystem to `~/.ssh/` and sets `chmod 600`
- Copies `ansible.cfg` to `~/` so it is not ignored (WSL mounts `/mnt/c/` as world-writable)
- Sets `ANSIBLE_CONFIG` and runs `ansible-playbook site.yml`

For re-deployments after a code push:

```bash
./deploy.sh --tags app
```

## Inventory

The inventory and group_vars are **generated by Terraform** — do not edit them by hand:

| File | Source |
|---|---|
| `inventories/vps/hosts.ini` | `../terraform/inventory.tpl` |
| `inventories/vps/group_vars/all.yml` | `../terraform/group_vars.tpl` |

Both files are git-ignored (see root `.gitignore`).

## Roles

| Role | What it does |
|---|---|
| `common` | apt upgrade, base packages, swap (2 GiB), sysctl tuning |
| `k3s` | Installs k3s (single-node, Traefik disabled) + Helm binary; copies kubeconfig for `ubuntu` user |
| `app` | Clones repo, runs `helm upgrade --install` from `helm/`, waits for Caddy on port 80 |

## Tags

Run a subset of the playbook with `--tags`:

```bash
# Full provisioning (first run)
ansible-playbook -i inventories/vps/hosts.ini site.yml

# Re-deploy / upgrade Helm release only (e.g. after a code push)
ansible-playbook -i inventories/vps/hosts.ini site.yml --tags app

# Re-install k3s + Helm only
ansible-playbook -i inventories/vps/hosts.ini site.yml --tags k3s

# Seed user (alice/bob/charlie - password123)
ansible-playbook -i inventories/vps/hosts.ini site.yml --tags seed
```

## Variables

Override any default in `inventories/vps/group_vars/all.yml` (generated)
or create `inventories/vps/host_vars/<ip>.yml` for host-specific overrides.

Key variables:

| Variable | Default | Description |
|---|---|---|
| `app_dir` | `/opt/banking-demo` | Remote path where the repo is cloned |
| `app_repo` | *(set me)* | Git remote URL |
| `app_repo_version` | `golang` | Branch / tag to deploy |
| `swap_size_mb` | `2048` | Swap file size in MiB |

## Dry run

```bash
ansible-playbook site.yml --check --diff
```
